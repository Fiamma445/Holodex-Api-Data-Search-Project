# Troubleshooting Log

이 문서는 HoloProject를 라이브 서비스로 운영하면서 실제로 겪은 문제와 해결 방향을 기록합니다. 현재 구조의 핵심은 외부 API를 매 화면마다 직접 호출하지 않고, 서버 동기화와 DB-backed API로 안정적인 탐색 경험을 만드는 것입니다.

## Issue 1. API Key가 없으면 라이브/예정 탭을 사용할 수 없던 문제

### Situation

초기 구현에서는 사용자가 Holodex API Key를 브라우저에 입력하지 않으면 라이브/예정 화면이 비어 보이거나 로딩 상태에 머무는 문제가 있었습니다. 하지만 라이브/예정은 공용 서비스의 기본 기능이므로, 사용자 개인 API Key가 없어도 동작해야 했습니다.

### Cause

프론트엔드가 사용자 localStorage의 API Key 존재 여부에 지나치게 의존했습니다. 서버에는 운영용 `HOLODEX_API_KEY`가 이미 설정되어 있었지만, 일부 경로가 이 서버 키를 제대로 활용하지 못했습니다.

### Fix

공용 기능은 서버 환경변수의 `HOLODEX_API_KEY`를 사용하도록 정리했습니다. 사용자는 별도 키 없이 라이브/예정, 아카이브, 노래 DB, 통계를 사용할 수 있습니다.

## Issue 2. 수동 동기화 버튼과 API Key 팝업이 사용자 흐름을 망치던 문제

### Situation

탤런트 관리 모달에 수동 동기화 버튼이 남아 있었고, 버튼을 누르면 관리자도 사용자용 API Key 입력 팝업을 보게 되는 혼란이 있었습니다. 실제 데이터 동기화는 서버에서 주기적으로 돌아가므로, 이 버튼은 현재 운영 구조와 맞지 않았습니다.

### Cause

초기에는 프론트엔드에서 사용자가 직접 동기화를 시작하는 흐름을 고려했지만, 운영 구조가 Railway 자동 동기화 중심으로 바뀐 뒤에도 UI 문구와 버튼이 남아 있었습니다.

### Fix

수동 동기화 버튼과 “동기화해야 반영됩니다” 문구를 제거했습니다. 탤런트 추가/삭제는 브라우저의 표시 목록에 즉시 반영되고, 검색/통계 데이터는 서버 DB의 1시간 자동 동기화를 기준으로 유지됩니다.

## Issue 3. 선택 필터 UI가 과하게 칠해지고 조합 조건이 어긋난 문제

### Situation

체크박스형 필터에서 여러 선택지를 조합했을 때 결과 조건이 기대와 다르게 해석되거나, 선택한 라벨 주변이 비정상적으로 칠해지는 UI 문제가 있었습니다.

### Cause

선택값 조합을 포함 조건으로 해석하는 로직이 명확하지 않았고, 체크박스 기본 선택 스타일과 커스텀 라벨 스타일이 충돌했습니다.

### Fix

선택한 항목을 포함 조건으로 해석하도록 정리하고, 체크박스와 라벨의 선택 상태가 과하게 칠해지지 않도록 스타일을 분리했습니다.

## Issue 4. API 과다 호출과 배포 자산 비대화 문제

### Situation

아카이브, 노래, 통계처럼 반복 조회가 많은 화면까지 외부 API에 직접 의존하면 Rate Limit, 응답 지연, 장애 전파 문제가 커질 수 있었습니다. 또한 seed DB와 채널 아이콘을 저장소나 런타임에 직접 싣는 방식은 배포 크기와 복구 시간을 늘렸습니다.

### Cause

서비스가 단순 API viewer에서 데이터 서비스로 커졌는데, 운영 자산 분리와 read model 설계가 초기 구조에 충분히 반영되지 않았습니다.

### Fix

주요 기능을 DB-backed API로 분리했습니다. 서버는 Holodex 데이터를 주기적으로 받아 SQLite에 적재하고, 프론트엔드는 `/api/search`, `/api/songs`, `/api/stats/*`, `/api/channel-index` 같은 내부 API를 사용합니다. 대용량 seed DB와 채널 아이콘은 R2/object storage로 분리해 `SEED_DB_URL`과 `assets.holo-search.xyz` 경로로 운영합니다.

## Issue 5. 채널 인덱스 초기 로딩이 불필요하게 무거웠던 문제

### Situation

탤런트 목록과 아이콘은 모든 화면에서 필요하지만, 매번 비싼 조회와 이미지 fallback을 반복하면 첫 화면 체감 속도가 나빠졌습니다.

### Cause

채널 메타데이터와 아이콘 경로가 화면 렌더링 중 여러 경로에서 반복 계산되었습니다.

### Fix

채널 인덱스 캐시와 정적 아이콘 경로를 분리했습니다. DB 상태가 바뀌지 않으면 서버는 캐시된 채널 인덱스를 반환하고, 프론트엔드는 R2 기반 아이콘 URL을 우선 사용합니다. 이 덕분에 초기 렌더링의 네트워크 흔들림을 줄였습니다.

## Issue 6. 노래 DB 최신성에 대한 오해

### Situation

특정 멤버의 최신 우타와꾸가 아카이브에는 보이는데 노래 DB에는 곡목이 바로 추가되지 않는 상황이 있었습니다.

### Cause

HoloSearch는 자체 음원 인식으로 곡을 추출하지 않습니다. Holodex/Musicdex가 `songs` 메타데이터를 붙인 뒤에야 다음 동기화에서 `video_songs`에 반영됩니다.

### Current Boundary

현재 자동 동기화 주기는 1시간입니다. 원천 서비스에 곡 정보가 추가되면 다음 동기화에서 반영될 수 있지만, 원천 메타데이터가 아직 없으면 HoloSearch도 보여줄 수 없습니다. 다음 개선으로는 최근 `singing` 영상 재조회 큐를 두어 뒤늦게 붙는 곡 메타데이터를 더 안정적으로 회수할 수 있습니다.
